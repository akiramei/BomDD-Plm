DESIGN DIRECTION(設計意図 — 抽出時は designIntent として扱う)

原則

  • 読み取り専用の検査工具。編集アフォーダンスを一切見せない(ボタン=フィルタ・表示切替のみ)
  • severity 色言語: error=赤 / warn=橙 / info=藍 / suppressed=灰+打消線。全ビュー共通
  • family 色言語: REQ=slate / E=青 / M=緑 / K=紫 / CP=橙 / SB=青緑。チップとグラフノードで同一
  • 密度は工具寄り(13px 基準)。数値は tabular-nums で桁ブレ防止

レイアウト不変条件(全ビュー)

  • topbar/viewtabs は固定。スクロールは main のみ(二重スクロール禁止)
  • サマリカードは固定幅 — 件数が 1桁でも4桁でも後続要素が動かない
  • 詳細パネル(グラフ)は固定幅 300px — 品目名長がグラフ領域を潰さない
  • パスは固定幅セル内で中央(先頭側)省略 — 長大パスが行高・列幅を変えない

状態の完全性

  • 空状態(所見ゼロ)は達成表示(「現ゲートで所見なし」)であり空白ではない
  • suppressed は非表示にせず灰色で残す(理由必須) — 黙って消えない
  • superseded 品目は灰+破線。そこへの active 参照(R-040)は赤エッジ+コールアウト
